## User

v0.dev の雰囲気を意識して、AI技術デモサイトを作ってください。

【参考にしたいv0の要素】
- 中央に大きなフォーカスエリア
- 「触ってみたい」と思わせる体験設計
- チャット欄と結果表示の明確な関係性
- 動きと変化が分かりやすい
- AIライブデモ感が強い

【ページ構成】
1. シンプルなヘッダー
   - センター寄せの「AI Live Demo」タイトル
   - サブタイトル「最新AI技術の統合実装例」

2. ヒーロー：DEMO STAGE
   - 画面中央に大きく配置
   - 訪問者の操作で変化する場所だと一目で分かる
   - 「ここがチャットで変化します」のプレースホルダー

3. 入力＋ログ
   - 下に大きめのチャット入力欄
   - その下にCONSOLE（折りたたみ可能）

4. サイドまたは下部
   - 使用技術バッジ
   - Skills の中身が見られるリンク

【デザイン指定】
- ダーク基調
- ベース色：#0a0a0a
- アクセント：エメラルド系（#10b981 など）
- フォント：Inter / JetBrains Mono
- 大きめのテキストとボタン
- アニメーションは控えめだが、入力時に反応がある感じ

【避けたいこと】
- 情報過多
- セクションの詰め込み
- カラフルすぎる配色

## User

デモエリア内の各要素に、ホバー時の要素名表示機能を追加してください。

仕様:
- デモエリア内の主要要素（タイトル、サブテキスト、ボタン等）にマウスを乗せると
- その要素が薄くハイライトされ
- 要素の隅または上部に小さなラベル（要素名）がツールチップ風に表示される
- マウスを離すとラベルは消える
- ラベルの表記例：「タイトル」「サブテキスト」「ボタン」など日本語

デザイン:
- ラベルは控えめに（小さめフォント、半透明背景）
- 全体のクリーンな見た目を損なわないこと
- ホバー時のハイライトはエメラルド系の細い枠で

スマホ対応:
- ホバーがない環境では、この機能は無効でOK（プロンプト例ボタンで代替）
